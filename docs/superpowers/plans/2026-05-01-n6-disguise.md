# N6 实施计划：盲盒身份 + 打听识破

> 起草日期：2026-05-01
> 分支：`refactor/n6-disguise`
> 依赖：N4 客人系统 + N5a 已合入

## 目标

让怪客真正"诡异"——来访时**不显示真名**，玩家面对的是"陌生人"，需消耗灵石（v1 简化）"打听"才能识破真身。
spec §6.5 + §7.3 的核心爽点：「玩家在线时收到识破检定提示」。

不在 N6 内：
- 打听检定的失败概率（v1 打听一定成功）
- 打听道具（v1 用灵石；后续做"打听签"）
- 离线按铺规自动放行（属 N5b）
- 怪客篡改真实数据（伪 tier 进结算）

## 设计要点

### 数据层
- `CustomerData` 加两字段：
  - `disguise_name: String`（伪名，空 = 真名公开 = 不伪装）
  - `disguise_tier: int`（伪装看起来的 tier，默认 -1 = 跟真实一致）
- `CustomerRequest` 加 `unmasked: bool = false`
- 现有 3 客人补伪装：
  - `meng_mian_ke`（蒙面客，tier=WEIRD）→ disguise_name = "陌生剑客"，disguise_tier = REGULAR
  - 其他 2 个常客不动（disguise_name = ""）

### 行为
1. CustomerSpawner.spawn_one：req.unmasked = (CustomerData.disguise_name 为空)
   - 真常客 unmasked = true（永远公开）
   - 伪装怪客 unmasked = false（盲盒）
2. CustomerArrivalPanel.show_request：
   - 若 unmasked：显示真名 + 真实 tier 标签
   - 否则：显示 disguise_name + 伪 tier 标签 + "打听（50 灵石）"按钮可见
3. 打听按钮按下：
   - 灵石不足 → 弹个提示（v1 简化用 push_warning，UI 只让按钮 disabled）
   - 灵石够 → 扣 50，req.unmasked = true，重渲染 panel
4. 借/拒按 req 真实身份（CustomerData.tier）走结算——无论是否打听过

### 兼容
- 既有 N4 测试不动（spawn_one 仍返回 CustomerRequest，只是多了 unmasked 字段；默认 false 但常客 disguise_name="" 自动设 true）
- 存档不需迁移——CustomerRequest 是运行时对象，不进存档
