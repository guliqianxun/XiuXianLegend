## 1. 世界观

天道崩坏后，灵气与"异象"混合。修士吐纳灵气=吸收污染，每一次突破都是力量与道心的赌局。
传统九境废止，修士被神秘侧"序列"收编：序列 9 入门，序列 0 触及神位。
玩家身份：**问道宗外勤修士**，奉命外出收容、镇压、研究异常。

**调性参考**：诡秘之主 + 一百诡夜 + 雾山五行美术 + 血源音效。

## 2. 核心循环

```
[白天] 挂机修炼 ──产出──▶ 经验 / 灵石 / 见闻 / 污染
   │
   ▼
[黄昏] 回城结算
   ├─ 锻造 / 洗练词缀（消耗灵石）
   ├─ 扮演突破（消耗见闻 + 道心检定）
   └─ 净化道心（消耗灵石）
   │
   ▼
[夜晚] 怪谈事件（事件文本 → 卡牌战斗 / 选择检定 → 掉落）
   │
   ▼
卡关 ──▶ 钻研某序列道途 ──▶ 解锁新流派 ──▶ 回到挂机
   │
   ▼
[赛季末] 主神归位 → 序列重置 → 保留"非凡日记"传承
```

## 3. 序列体系（首版 6 道途）

每条道途 10 阶（序列 9 → 序列 0）。

| 道途 | 主题 | 标志能力 | 反噬 |
|---|---|---|---|
| **剑** | 物理斩击 / 暴击 | 剑域、御剑 | 嗜血、剑鸣 |
| **咒** | 符箓 / 持续伤害 | 燃烧、诅咒、封印 | 反噬、心魔 |
| **傀** | 召唤 / 控场 | 傀儡军团、提线 | 反控、寄生 |
| **丹** | 增益 / 治疗 | Buff / 解控 | 药毒、依赖 |
| **食** | 吞噬 / 转化 | 吸血、夺词缀 | 异化、饥饿 |
| **卜** | 预判 / 概率操控 | 必中、规避、改判 | 因果反噬 |

**突破=扮演仪式**：消耗见闻 + 特定异常物，进行道心检定。失败：失控（短期）/ 暴毙（极小概率，赛季末复活）/ 异化（永久负面 Buff，可解）。

## 4. 战斗系统

- **自动回合制 + ATB 速度条**，可 1×/2×/4× 倍速、跳过。
- **卡组**：30 张上限，从已学技能池中编。
- **资源**：灵气（每回合恢复）+ 道心（每场战斗的"血量"，归零进入心魔阶段）。
- **污染牌**：强力但每次出牌涨疯狂值；疯狂值满进入失控（敌我不分）。
- **流派加成**：清醒流（道心>70% 增伤）/ 疯狂流（道心<30% 增伤）/ 平衡流（切换增益）。

## 5. 装备 / 词缀（暗黑核心）

**6 件套**：剑、符、傀核、丹炉、食器、卦盘。
**品质**：凡（白）/ 灵（绿）/ 法（蓝）/ 禁（紫）/ 秘（金）。

| 品质 | 词缀槽 | 特性 |
|---|---|---|
| 凡 | 1-2 | 数值小 |
| 灵 | 2-3 | 数值中 |
| 法 | 3-4 | 数值大 |
| **禁** | 4-5（含 1 反噬） | 高数值 + **强制负面词缀** |
| **秘** | 5（含传说词缀） | 需"封印"才能装备，封印消耗道心上限 |

**词缀池**（首版 ~60 条，正/负各半）：
- 正：攻+X%、暴击+X%、灵气恢复+X、闪避+X%、出某色卡时+X% 伤害…
- 负：午夜后攻击+30% 但每回合 5% 自伤、敌人死亡时 20% 概率污染+5、装备时道心-10…
- 套装：3/5 件激活（如"剑·疯狂套" 5 件：道心<30% 时所有剑系卡费用-1）。

**洗练**：消耗灵石重铸全部词缀（不保底）。**圣化**：消耗稀有材料锁定一条词缀。

## 6. 挂机产出（核心数学）

离线时间 → 一次性公式结算（**不模拟回合**）：

```
经验 = base_xp(stage) × time_h × (1 + 修炼加成)
灵石 = base_gem(stage) × time_h × (1 + 财运加成)
见闻 = poisson(λ = time_h × 见闻率)
污染 = base_pollute × time_h × (1 - 抗污染)
掉落 = 离线掉落表抽样（按 time_h 决定抽样次数，封顶）
```

**封顶设计**：单次离线最多结算 12 小时；超过部分按 50% 衰减结算 12-24h，24h 后丢弃。
**反通宵**：每日首次结算 +20%，反向激励"留人"而非"挂机即胜"。

## 7. 赛季机制

- 一季 60 天。
- 季初：抽取一名"主神"，决定本季词缀池偏移、Boss、序列特性。
- 季末：序列进度重置，保留"非凡日记"（永久解锁的少量传承条目，如 1-2 个词缀偏好、1 个起手卡）。
- 赛季排行榜（本地+可选云同步）。

## 8. 设计模式与资源约束（**重要**）

> 目标：**单机包体 < 500MB，运行时内存 < 800MB，避免任何"资源过载"陷阱**。

### 8.1 强制遵循的模式

| 模式 | Godot 落地 | 用途 |
|---|---|---|
| **数据驱动 (Resource)** | `class_name CardData extends Resource` 等，存为 `.tres` | 卡 / 装 / 词缀 / 序列 / 怪谈事件全部 Resource，**禁止硬编码进脚本** |
| **服务定位器 / Autoload 单例** | `EventBus / GameState / SaveSystem / DataRegistry / AudioBus` | 全局唯一入口，避免节点路径耦合 |
| **信号事件总线** | EventBus 自动加载 + 全局 signal | UI 与逻辑解耦；战斗结算 → UI 飘字走信号 |
| **状态机** | combat 阶段、敌我单位、UI 模态 | 用 `GDScript` 简单 FSM，不引第三方插件 |
| **命令模式** | 卡牌效果 = `CardEffect` 子类列表 | 可组合 / 可序列化（用于回放、调试） |
| **对象池** | 飘字、粒子、卡牌实例、伤害数字 | **战斗内任何会重复 instance 的节点必须走池**；池满则复用最旧 |
| **观察者** | Godot signal 原生 | 避免轮询 |
| **MVVM** | View 只读 ViewModel，不直接读 Model | UI 节点禁止 import 业务模型 |

### 8.2 资源加载规则

- **场景仅在切入时加载**：`ResourceLoader.load_threaded_request()`，配 loading 屏。
- **图集**：所有 ≤256² 图标进 TextureAtlas，单图禁止 >1024²（卡面除外，卡面 768×1024）。
- **音频**：BGM 用 `.ogg` 流式；SFX 用 `.wav` 但单文件 < 200KB，优先复用。
- **DataRegistry 启动时只加载 _索引_**，具体 Resource 按需 `load()`，加载后进 LRU 缓存（cap=200）。
- **存档**：JSON，差量写入；版本号 + migration 函数列表，禁止破坏旧存档。

### 8.3 性能预算（硬指标）

- 战斗场景节点数 ≤ 300。
- 每帧脚本耗时 ≤ 4ms（60fps 下留余量）。
- 启动到主城 ≤ 3 秒（SSD）。
- 离线结算 1 年的存档 ≤ 100ms 计算完毕（纯公式，无循环模拟）。

### 8.4 反例 / 禁止事项

- ❌ 不要为每个词缀写一个 Node，**词缀是数据，不是节点**。
- ❌ 不要在 `_process` 里查找节点（`get_node` / `find_child`）。
- ❌ 不要把战斗逻辑塞进 UI 脚本。
- ❌ 不要引入 ECS 框架（GDScript 上性价比低）；用 Node + Resource + 信号已足够。
- ❌ 不要写"以后可能用到"的抽象层；YAGNI。

## 10. 命名约定

- 文件名：`snake_case.gd / .tres / .tscn`
- 类名：`PascalCase`，必须 `class_name`
- Resource 文件目录：`data/<category>/<id>.tres`，id 全小写下划线
- Signal：过去时（`card_played`, `enemy_died`）
